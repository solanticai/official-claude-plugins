## 2. Findings by Domain

### 2.1 Security (1 CRITICAL, 1 HIGH)

#### AA-001 — Service-role key exposed to browser bundle (CRITICAL)
- **File:** `apps/web/src/lib/supabase/admin.ts:4`
- **Evidence:** `import { createClient } from '@supabase/supabase-js'` followed by `createClient(url, process.env.SUPABASE_SERVICE_ROLE_KEY!)`. The file is imported by `apps/web/src/components/AdminPanel.tsx:3`, which is a client component (line 1 has `'use client'`).
- **Why it matters:** The bundler will inline the service-role key into the client JavaScript, granting any visitor full RLS bypass. This is the highest-impact finding in the run.
- **Remediation:** Move the admin client to a server-only module (`apps/web/src/lib/supabase/admin.server.ts`) and call it from a Server Action or Route Handler. Rotate the service-role key in the Supabase dashboard immediately — assume it is compromised.
- **Verification:** Run `pnpm build && grep -r SUPABASE_SERVICE_ROLE .next/static`. The grep must return nothing.

#### AA-006 — Storage bucket policy allows anonymous read on `user-uploads` (HIGH)
- **File:** Supabase MCP `list_storage_policies` output
- **Evidence:** Bucket `user-uploads` has policy `allow_anon_read` that grants `SELECT` to `anon`.
- **Remediation:** Replace with an authenticated-user policy keyed by `auth.uid() = (storage.foldername(name))[1]::uuid`.

### 2.2 Postgres (1 HIGH, 1 MEDIUM)

#### AA-002 — Prepared statements in Supavisor transaction mode (HIGH)
- **File:** `apps/web/src/lib/db/orders.ts:18,42`
- **Evidence:** Drizzle queries use `prepare()` and the connection string in `apps/web/src/lib/db/pool.ts:7` is the `:6543` Supavisor transaction-mode endpoint.
- **Why it matters:** Supavisor in transaction mode does not support prepared statements; queries will fail intermittently under load.
- **Remediation:** Either drop `prepare()` (Drizzle parametrises queries either way), or switch the connection to the `:5432` direct URL behind a server-only client.
- **Verification:** Load-test with `k6` against `/api/orders`; run `SELECT * FROM pg_stat_statements WHERE query LIKE 'PREPARE%'` — should return zero rows after the fix.

#### AA-007 — `orders.created_at` lacks a btree index (MEDIUM)
- **File:** `supabase/migrations/20251110_create_orders.sql`
- **Evidence:** `pg_stat_statements` shows `SELECT ... FROM orders ORDER BY created_at DESC LIMIT 50` running 4,200 times/day with mean duration 280 ms.
- **Remediation:** `CREATE INDEX CONCURRENTLY idx_orders_created_at ON orders (created_at DESC);`
- **Verification:** Re-run the same `pg_stat_statements` query 24 h after deployment; mean duration should drop below 20 ms.

### 2.3 Server/Client (1 HIGH)

#### AA-003 — Auth-aware page caches via `revalidate=3600` (HIGH)
- **File:** `apps/web/src/app/dashboard/page.tsx:1`
- **Evidence:** `export const revalidate = 3600` at module top; the page calls `cookies()` and renders user-specific data fetched via `getUser()`.
- **Why it matters:** Next.js will serve the cached HTML to *any* user, leaking the cached user's session-keyed data.
- **Remediation:** Either set `export const dynamic = 'force-dynamic'` or remove `revalidate` and rely on per-request rendering for auth-aware pages.

### 2.4 Leak Detection (1 HIGH)

#### AA-004 — Realtime channel never unsubscribed (HIGH)
- **File:** `apps/web/src/components/LiveOrders.tsx:24`
- **Evidence:** `useEffect(() => { supabase.channel('orders').on(...).subscribe() })` — no cleanup function returned.
- **Remediation:** Return `() => channel.unsubscribe()` from the effect.

### 2.5 Connection Limits (1 MEDIUM)

#### AA-005 — Pool sizing leaves 5% headroom (MEDIUM)
- **File:** `apps/web/src/lib/db/pool.ts:12`
- **Evidence:** `max: 50` per Vercel function instance × 4 concurrent functions = 200; PostgREST reserves 15; plan limit 200 (Supabase Pro + small compute, confirmed in resolved question).
- **Remediation:** Drop pool `max` to 30 per instance. Monitor via Supabase dashboard's connection chart for one week.

### 2.6 Frontend (3 MEDIUM)

- **AA-008** — Three client components import `framer-motion` at top level — adds 38 KB to first-load JS. Move to `next/dynamic` with `ssr:false`.
- **AA-009** — `<Image>` in `Hero.tsx:8` lacks `priority`; LCP element will lazy-load.
- **AA-010** — `tailwind.config.ts` `content` glob misses `apps/web/src/components/marketing/**` — three components ship unstyled in production.

### 2.7 Other (2 LOW, 1 INFO)

- **AA-011** (LOW, bug-finder) — `apps/web/src/app/api/orders/route.ts:14` swallows error from `await req.json()` with empty catch.
- **AA-012** (LOW, backend) — Edge Function `apps/web/supabase/functions/billing/index.ts` has no `CORS` headers; only called server-side today but worth fixing before browser use.
- **AA-013** (INFO, client-connection) — Browser Supabase client instantiated at module scope; consider `useRef`-based instantiation if multiple components import it.
