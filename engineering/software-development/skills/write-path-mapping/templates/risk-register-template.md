- **Rationale:** These leak data across security boundaries. Fix before production.

- **Checklist:** Add auth middleware; create RLS policies; add tenant filters; verify with tests.

### Batch 2 — Webhook Verification / Service-Role Overreach (CRITICAL)
- **Rationale:** Inbound untrusted data paths must verify signatures; service-role keys must never trust user input.
- **Risks:** {{n}}
- **Checklist:** Implement HMAC verification; narrow service-role scope; add defensive checks.

### Batch 3 — Transactions & Race Conditions (HIGH)
- **Rationale:** Partial writes corrupt data. Wrap multi-target writes in a transaction.
- **Risks:** {{n}}
- **Checklist:** Identify multi-target handlers; wrap in DB transaction or saga.

### Batch 4 — Cache Invalidation & Orphan Queues (HIGH)
- **Rationale:** Stale caches and orphaned queues cause silent drift.
- **Risks:** {{n}}
- **Checklist:** Pair every DB write with a matching cache invalidation; resolve queue consumers.

### Batch 5 — Idempotency & Audit (MEDIUM)
- **Rationale:** Retries and compliance.
- **Risks:** {{n}}
- **Checklist:** Add idempotency keys to webhook/queue consumers; audit-log sensitive writes.

### Batch 6 — Input & Upload Hardening (MEDIUM)
- **Rationale:** Resource abuse prevention.
- **Risks:** {{n}}
- **Checklist:** Cap array sizes; enforce MIME types; enforce size limits; add timeouts.

### Batch 7 — External API Resilience (INFO → MEDIUM)
