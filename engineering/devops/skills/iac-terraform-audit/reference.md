## Â§1 â€” Audit Taxonomy

### A. State & Backend
| ID | Check |
|---|---|
| A.1 | Remote backend declared (S3/GCS/azurerm/HTTP, not `local`) |
| A.2 | Backend encryption at rest (SSE, KMS key ref) |
| A.3 | State locking (DynamoDB table for S3, GCS native, azurerm native) |
| A.4 | Workspaces or directory-per-env separation |
| A.5 | `.terraform/` and `*.tfstate*` in `.gitignore` |
| A.6 | State bucket versioning enabled |
| A.7 | Cross-account access for state buckets follows least-privilege |

### B. Provider pinning
| ID | Check |
|---|---|
| B.1 | `required_providers` block declares every provider used |
| B.2 | Every provider has a version constraint (`~>` or exact) |
| B.3 | `required_version` for Terraform/OpenTofu set |
| B.4 | Provider `source` is fully qualified (`hashicorp/aws`, not `aws`) |
| B.5 | Lockfile (`.terraform.lock.hcl`) committed |

### C. Security (Checkov/tfsec taxonomy)
| ID | Check |
|---|---|
| C.1 | No `0.0.0.0/0` in security group ingress for non-public ports |
| C.2 | No public S3 bucket ACLs unless explicit static-site suppression |
| C.3 | Encryption at rest on RDS, EBS, EFS, S3, DynamoDB, SNS, SQS |
| C.4 | No IAM policies with `Action: "*"` + `Resource: "*"` + `Effect: Allow` |
| C.5 | No plaintext secrets in `*.tfvars` or `variables.tf` defaults |
| C.6 | KMS keys have rotation enabled |
| C.7 | CloudTrail / audit logging enabled on critical accounts |
| C.8 | Database backups / snapshots configured |

### D. Module hygiene
| ID | Check |
|---|---|
| D.1 | Every `variable` has `type` |
| D.2 | Every `variable` has `description` |
| D.3 | `validation` blocks on bounded inputs |
| D.4 | `sensitive = true` on secret-bearing variables and outputs |
| D.5 | Every `output` has `description` |
| D.6 | Defaults are safe (no `allow_public = true` by default) |
| D.7 | Module has a README with input/output table (often generated by `terraform-docs`) |

### E. Environment separation
| ID | Check |
|---|---|
| E.1 | Prod and non-prod in separate states |
| E.2 | No cross-environment `terraform_remote_state` refs |
| E.3 | Workspace naming convention is consistent |
| E.4 | Environment-specific `.tfvars` files clearly labelled |

### F. Drift risk
| ID | Check |
|---|---|
| F.1 | `lifecycle.ignore_changes` targets specific attributes, not `all` |
| F.2 | `null_resource` + `local-exec` usage justified in comments |
| F.3 | `create_before_destroy = true` on resources that require blue/green |
| F.4 | No `terraform import` drift â€” live plan clean |

### G. Cost hotspots
| ID | Check |
|---|---|
| G.1 | Default instance types not oversized (`t3.medium` default not `m5.4xlarge`) |
| G.2 | Autoscaling max bounded |
| G.3 | NAT Gateway cost acknowledged (one per AZ can be $100+/month) |
| G.4 | Unattached EIPs / unused ELBs detected |
| G.5 | `log_retention_in_days` set on CloudWatch log groups (unbounded = expensive) |

### H. Testing & CI
| ID | Check |
|---|---|
| H.1 | `terraform fmt -check` in CI |
| H.2 | `terraform validate` in CI |
| H.3 | `tflint` or `tfsec` or `checkov` in CI |
| H.4 | `terraform-docs` generation in CI |
| H.5 | Pre-commit hooks (`pre-commit-terraform`) configured |

## Â§2 â€” Severity Rubric

| Severity | Examples |
|---|---|
| CRITICAL | Plaintext secret in tfvars, `0.0.0.0/0:22` ingress, public S3 with PII, IAM `Action: "*"` on prod |
| HIGH | No remote backend (state local), missing provider pin on prod module, missing encryption on RDS |
| MEDIUM | Missing variable descriptions, `lifecycle.ignore_changes = all`, no CI lint |
| INFO | Cost hotspots, naming convention mismatches, missing README |
